BlueTrace · UI 原型 v3 iOS 适配

与 v3_android.html 同一套设计 token、同一套传感器配色、同一套数据可视化——这就是「统一设计语言」。 分叉只发生在系统外壳：导航栏（返回箭头 + 标题 / 大标题）、底部 Action Sheet、iOS 开关与分段控件、 灵动岛 + 底部 Home Indicator 安全区、一次性系统权限弹窗、以及后台 Core Bluetooth 机制。 下方只展示真正分叉的关键屏；未列出的屏（传感器总览各卡、波形视图、结束摘要细节等）与 Android 版视觉完全一致。

同一设计语言 HIG 平台自适应 Action Sheet / 导航栏 / 安全区 Core Bluetooth 后台

📐 iOS 自适应清单（对照 Android 版）

① 顶部 App Bar → 导航栏（左上返回箭头 + 居中标题；列表页用大标题）。② Bottom Sheet → Action Sheet（破坏项红色置底 + 独立"取消"）。③ Material 开关/分段 → iOS Toggle / Segmented Control。④ FAB/底部按钮 → iOS 大圆角底部按钮，无 FAB。⑤ Snackbar → 顶部胶囊 Toast。⑥ 底部留 Home Indicator 安全区。⑦ 运行时逐项权限 → 一次性系统弹窗 + Info.plist 用途文案。⑧ 前台服务保活 → bluetooth-central 后台模式 + 状态恢复（详见 architecture/BlueTrace_CrossPlatform_Notes.md）。

① 启动 / 权限（iOS 系统弹窗）

iOS 权限是一次性系统弹窗，弹窗文案来自 Info.plist；BlueTrace 自绘的"用途说明层"在系统弹窗前出现。iOS 蓝牙扫描不需要定位权限。
