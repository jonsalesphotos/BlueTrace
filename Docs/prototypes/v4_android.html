第四版信息架构：主界面改为底部三 Tab 中枢 —— 采集 / 数据 / 设置。 采集 是 BlueTrace 自有的入口中枢，汇集 设备 · 采集用户 · 采集模式，底部双 CTA 进入扫描连接 / 开始采集； 数据 收纳采集会话（D-6 一会话一文件夹）；设置 放环境权限 / GNSS / 存储 / 关于。 视觉沿用 v3 浅蓝 Material 3 token；本轮经设计面板精修：字阶收紧（标题 19、卡标题 14、三级字重 700/600/400），密度对齐 4/8 网格，采集主界面一页放下不滚动。

主界面 · 底部三 Tab 中枢

三个 Tab 是 App 的主舞台。采集 Tab 为主界面：点入口卡进子页（扫描连接 / 采集用户 / 采集模式），底部「开始采集」进采集运行界面。子页与采集运行页进入后 隐藏底部 Tab，避免强状态任务被误触切走。精修要点：层级靠对比与字重而非字号；去掉装饰性虚线分隔；采集模式二选一收进尾部分段控件；双 CTA 沉底，留弹性呼吸留白。

启动与权限

流程上在主界面之前（此处置于主界面后展示）。启动屏一闪而过，背后做静默环境检查：首次启动弹权限请求；后续启动静默检查、仅缺权限才弹。权限可跳过进主界面，用到时再就地请求该权限（系统永久拒绝则引导去应用设置页）。硬性权限（BLE 扫描/连接）缺则在扫描/开始采集处拦截；建议权限（通知、定位/GNSS）缺不阻断。蓝牙开关 ≠ 权限，单独检测与处理。

横切状态（采集中 / 暂停重连 / 已停止 + 运行日志）

专业采集工具，异常不发散。运行只有三种状态：🟢 采集中 · 🟡 暂停·重连中（连接断开 → 自动重连，原始 HEX 日志续写不丢数据）· 🔴 已停止（存储满等不可继续）。无法处理的坏包 / CRC 错 / 解码失败 / 重连记录一律进运行日志、不打断采集。进程被回收可恢复；运行中离开时采集 Tab 顶部常驻"正在采集"提示条。
