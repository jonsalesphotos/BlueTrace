# BlueTrace 交互 / UX 规格

> **这份文档回答"怎么交互"**：信息架构、页面流、状态机、边界情况，以及 Android / iOS 的少数分叉点。
> 需求见 [BlueTrace_PRD.md](BlueTrace_PRD.md)，视觉见 [BlueTrace_Design_System.md](BlueTrace_Design_System.md)，可视化原型见 [../prototypes/](../prototypes/)。

> **给新手**：交互文档的核心是**状态**，不是"页面"。一个页面在不同状态下长得不一样、能干的事也不一样。先把状态画清楚，页面跳转自然就出来了。

## 1. 信息架构（页面地图）

```text
启动
  └─ 静默环境检查
       ├─ 首次启动 / 硬性条件不完整 ─→ 权限与环境检查 ─(完成)─┐
       └─ 非首启且静默通过 ────────────────────────────────┤
                                                            ▼
                                              主界面 Home（工具中枢）
                                               ├─ 扫描并连接设备 ─→ 设备分配（会话准备）
                                               │                     ├─ 扫描 DUT / REFERENCE
                                               │                     ▼
                                               │               DUT 配置与控制
                                               │                ├─ 传感器总控（开关 + 采样率）
                                               │                ├─ 设备端算法控制
                                               │                └─ 传输与打包策略（批包/组帧/带宽）
                                               │                     ▼
                                               │               数据采集（运行时主屏）
                                               │                ├─ 多传感器总览
                                               │                ├─ 波形视图 ⇄ 原始分包流视图
                                               │                └─ 运行中控制面板（弹层）
                                               │                     ▼ （长按 2 秒）
                                               │               采集结束摘要
                                               ├─ 环境与权限检查（手动复查入口）
                                               └─ 历史会话 → 会话详情 → 导出

横切：异常 Banner（任意采集态）/ 进程恢复弹层（启动时）
```

> **页面可见性修订（2026-06-16）**
> - **权限页**：不再每次启动都弹。仅"首次启动"或"启动静默检查发现硬性条件不完整"时门控弹出；其余启动静默放行进入 Home。Home 提供"环境与权限检查"手动复查入口（也可后续放进设置页）。
> - **扫描页**：不再开屏即扫；仅由 Home 的"扫描并连接设备"按需进入。
> - **硬性条件阻断**（F-PERM-2）从"线性门"调整为"守在采集入口"：Home 的"扫描并连接设备"在环境未就绪时引导回权限页处理，符合"缺硬性条件阻断采集"的本意。

## 2. 全局导航模型

- **落地是"主界面 Home（工具中枢）"**（修订 2026-06-16）：启动先做静默环境检查，仅首次启动或硬性条件不完整时插入权限页，其余直接进 Home。Home 汇集"扫描并连接设备""环境与权限检查（手动）""历史会话"等入口。
- **主线是"流程式（向导）"**：Home → 扫描并连接设备 → 设备分配 → 配置 → 采集 → 结束，一步步向前；不是底部 Tab 平行切换（采集是强状态任务，不适合随意跳 Tab）。
- **权限页**：首次启动 / 硬性条件不完整时为门控（无返回）；从 Home 手动进入时可返回。
- **扫描页**：按需从 Home 进入，不再开屏即扫。
- **历史/文件**作为旁支入口（从 Home 或结束摘要进入）。
- **采集运行中**：导航被"软锁定"——返回不等于停止；离开采集页时全局显示"正在采集"提示条，点击可回。

> 平台差异见 §6。

## 3. 核心状态机

### 3.1 会话状态 SessionState
```text
Idle ──start()──> Preparing ──全部角色就绪──> Running ──stop()──> Ended
                                   │
                                   └──全局失败(蓝牙关/权限撤销)──> Failed
```

### 3.2 单角色运行态 RoleRuntimeState（每个设备各一份，互相独立）
```text
NotAssigned → Connecting → Subscribing → Collecting
                  ↑              │
                  └──Reconnecting┘（断线，AutoConnect 兜底）
Collecting/Connecting ──不可恢复错误──> Failed(reason)
```

**关键语义**：会话用 supervisorScope，**某个角色 Failed 不取消其它角色**。
- 单设备会话：只有一个角色，"部分失败"退化为"该设备失败"。
- 多设备会话：UI 显示如"REFERENCE 在采、DUT 重连中"是正常态。
- 仅当**全局事件**（蓝牙关闭、权限撤销）才整会话进入 Failed。

### 3.3 传感器路状态（DUT 内每路）
```text
Enabled(采样中) ⇄ Paused(运行中临时关) → Streaming(数据在流) → Stalled(单路中断)
```
单路 Stalled 只影响该路落盘，其余路继续。

## 4. 逐屏交互要点

> 屏号对应 [../prototypes/v3_android.html](../prototypes/v3_android.html) 与 v3_ios.html。

| 屏 | 关键交互 | 主要状态 |
| --- | --- | --- |
| 启动 | 静默环境检查 + 首启判断 + 可恢复会话 → 决定落地 Home 还是权限页 | 检查中 |
| 主界面 Home | 工具中枢：环境状态概览（可点进权限页）+ "扫描并连接设备" 入口 +（P2）采集会话 / 历史 | 就绪 / 待处理 |
| 权限检查 | 仅首启 / 不完整时弹（门控无返回）；可从 Home 手动进入复查（带返回）；硬性条件就绪才放行采集 | 缺项 / 就绪 |
| 设备分配 | 由 Home"扫描并连接设备"进入；点角色卡进扫描；DUT 卡显示已识别传感器；底部"下一步" | 空 / 单角色 / 双角色 |
| DUT 扫描 | Service UUID 过滤 + RSSI 排序；选中→连接 | 扫描中 / 有结果 / 无结果 |
| 传感器总控 | 逐路开关 + 采样率芯片；变更后"下发设备" | — |
| 算法控制 | 勾选设备端算法；下发 | — |
| 传输/打包策略 | 调批包大小/组帧周期，实时带宽预估与安全判定 | 安全 / 超带宽告警 |
| 采集总览 | 每路一卡；进入双视图/控制面板；底部长按结束 | 全采 / 部分降级 |
| 波形视图 | 顶部 segmented 切「波形/分包流」；多路波形大图 | — |
| 原始分包流 | 滚动批包(紫)/组帧(绿) Hex；包速率/丢包/CRC | — |
| 运行中控制面板 | 不停采集开关传感器/切算法；指令日志 → / ← ACK | — |
| 长按结束 | 环形进度 2 秒；松手取消 | 进行中 / 完成 |
| 结束摘要 | 每路文件 + 行数；导出/查看 | — |
| 历史/详情 | 展示传感器组合；勾选数据流导出 | — |
| 异常合屏 | 红/黄/蓝分级 Banner；单路异常不停其它路 | 各异常态 |
| 进程恢复 | 逐角色判断：可续采 / 需重扫 | 全恢复 / 部分 / 全重扫 |

## 5. 边界情况清单（每个都要有明确表现，不可静默失败）

| 场景 | 表现 | 行为 |
| --- | --- | --- |
| 采集中关蓝牙 | 红 Banner「采集暂停」 | 重开后自动续采 |
| 单路传感器中断 | 黄 Banner「IMU 流中断，其余在采」 | 该路重订阅，其余不动 |
| 参考设备断开 | 黄 Banner「REFERENCE 断开，DUT 不受影响」 | 仅该角色重连 |
| 存储空间不足 | 红 Banner「存储失败」 | 建议停止并导出 |
| 权限被撤销 | 回权限页 | 停相关能力，引导重授权 |
| 配置带宽超限 | 策略页告警 | 阻止"开始"，提示降采样率/增批包 |
| 误触返回键（采集中） | 弹"最小化/停止/取消" | 返回 ≠ 停止 |
| 进程被杀 | 启动时恢复弹层 | 逐角色续采或重扫 |

## 6. 平台差异（Android ↔ iOS）

> 统一设计语言下，**绝大多数屏两端一致**。以下是**真正分叉**的点——交互上要做平台自适应。组件级映射见 [Design System §平台映射](BlueTrace_Design_System.md#6-平台自适应映射androidios)。

| 维度 | Android | iOS |
| --- | --- | --- |
| 顶部导航 | 顶部 App Bar，标题居左/居中 | 导航栏：左上返回箭头 + 标题；可用大标题 |
| 返回手势 | 系统返回键 / 手势返回 | **左缘右滑返回**（需保证可滑） |
| 弹层 | Bottom Sheet | **Action Sheet**（破坏性操作红色置底） |
| 轻提示 | Snackbar（底部） | 顶部胶囊 Toast / 无 Snackbar 概念 |
| 主操作 | 可用 FAB 或底部按钮 | 底部按钮 / 导航栏右上操作，**不用 FAB** |
| 选择控件 | Material 开关、芯片 | iOS 开关、Segmented Control |
| 安全区 | 状态栏 / 导航栏 inset | 刘海/灵动岛 + 底部 Home Indicator inset |
| 权限弹窗 | 运行时逐项请求 | 一次性系统弹窗 + Info.plist 用途文案 |
| 后台采集 | 前台服务通知常驻 | 无常驻通知；系统态栏蓝条提示后台用蓝牙 |
| 进程恢复 | 启动恢复 + 通知引导 | CBCentralManager 状态恢复 + 启动恢复 |

**交互原则**：流程、状态、信息层级、配色、数据可视化**两端完全一致**；只有"系统级外壳"（导航条、弹层形态、安全区、权限/后台机制）按平台走。这样用户在任一平台都觉得"既是 BlueTrace、又像这台手机该有的样子"。

## 7. 关键交互规则（沉淀为开发统一依据）

1. 用户必须随时知道是否正在采集（运行中全局提示条）。
2. 采集中返回键/手势 ≠ 停止；停止必须明确确认（长按 2 秒）。
3. 至少一个设备连接成功才允许开始采集。
4. 单路/单设备异常不得中断其它路/设备的落盘。
5. 异常一律可见、分级（红=阻断/全局，黄=单路降级，蓝=可恢复）。
6. 设备控制指令即时生效并写入会话 manifest。
7. 采集结束必须有结果摘要；文件必须能被找到和导出。
8. 平台外壳自适应，产品语言统一。
