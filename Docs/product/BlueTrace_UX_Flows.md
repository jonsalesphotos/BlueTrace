# BlueTrace 交互 / UX 规格

> **这份文档回答"怎么交互"**：信息架构、页面流、状态机、边界情况，以及 Android / iOS 的少数分叉点。
> 需求见 [BlueTrace_PRD.md](BlueTrace_PRD.md)，视觉见 [BlueTrace_Design_System.md](BlueTrace_Design_System.md)，可视化原型见 [../prototypes/](../prototypes/)。

> **给新手**：交互文档的核心是**状态**，不是"页面"。一个页面在不同状态下长得不一样、能干的事也不一样。先把状态画清楚，页面跳转自然就出来了。

## 1. 信息架构（页面地图）

```text
启动 → 静默环境检查（+ 首启判断 + 可恢复会话）
  ├─（首启 / 硬权限缺失）→ 权限门控页（无返回）→ 完成 → 采集 Tab
  └─（否则）─────────────────────────────────→ 采集 Tab
        （可恢复会话 → 进程恢复弹层，逐角色续采/重扫）

底部三 Tab（常驻中枢 · 唯一主 IA）
├─ ① 采集 Tab（主中枢）
│   ├─ 设备入口 → 设备连接（扁平列表 · 单击连接/断开 · RSSI 过滤 · 连接计数 · DUT ≤3 + 参考心率带 ≤1）
│   ├─ 用户入口 → 用户选择 → 用户编辑（别名/性别/生日/身高/体重）
│   ├─ 采集模式（Wear ⇄ Unwear，内联分段）
│   ├─（设备连接后）传感器配置 → 传感器总控（纯开关：该路是否采集 · 透明传输 · 不控采样率）；设备端算法（后期）
│   │      ※ 打包策略页已删除（D-5：链路参数由设备驱动，App 只观测）
│   └─ 主 CTA「开始采集」→ 采集运行（全屏，隐藏 Tab）
│         ├─ 数据采集（照竞品 Data Collection）：设备卡(含 HR) + 采集类型勾选 + Datas/Time 简单实时数据区 + Start/End 标签
│         ├─ 暂停 / 长按 2 秒结束（唯一退出）
│         └─ 长按 2 秒结束 → 采集结束摘要（→ 数据详情 / 导出）
├─ ② 数据 Tab → Raw Data File 列表（按会话文件夹，D-6 · 搜索 · 多选）→ 会话详情 → 导出
└─ ③ 设置 Tab → 环境与权限检查 / GNSS 数据源开关 / 用户管理 / 导出位置 / 存储 / 关于
                （二期项：服务器/上传/实时透传 灰显标注）

横切：异常 Banner（任意采集态）/ 进程恢复弹层（启动时）/ "正在采集"全局提示条（离开运行页时）
```

> **IA 修订（V4 · 2026-06-16）**：落地与主架构改为**底部三 Tab（采集 / 数据 / 设置）**，取代 v3 向导式 Home 中枢（v3 降为 legacy 参考）。
> - **Tab 可见性**：仅三个顶级中枢显示底部 Tab；进入任一子页或采集运行时**隐藏 Tab**（push 页带返回），避免强状态任务被误切。
> - **权限页**：仅"首启 / 硬权限缺失"门控弹出；其余静默进采集 Tab。设置 Tab 提供"环境与权限检查"手动复查。
> - **硬性条件阻断**（F-PERM-2）守在"进入采集/扫描"入口：环境未就绪时引导回权限页。
> - **采集运行（V4 精简 · 照竞品 Data Collection 重做 2026-06-16）**：设备卡(含 HR) + 采集类型勾选 + 简单实时数据区 + Start/End 标签 + 暂停 + 长按 2 秒结束；运行头「用户 · Wear/Unwear · 计时」。已去波形⇄分包流双视图 / 只读观测 / 运行中控制面板。

## 2. 全局导航模型

- **落地与主架构 = 底部三 Tab（采集 / 数据 / 设置）**（V4，2026-06-16）：启动先做静默环境检查，仅首启或硬权限缺失时门控弹权限页，其余直接进**采集 Tab**。
- **三 Tab 平行切换**为常驻中枢；**采集是强状态任务**，从采集 Tab **向前 push** 推进（设备 → 用户/模式 → 配置 → 运行 → 摘要），过程中**隐藏底部 Tab**。数据 / 设置为旁支中枢。
- **权限页**：首启 / 硬权限缺失时门控（无返回）；从设置 Tab 手动进入可返回。
- **设备扫描**：从采集 Tab 的设备入口按需进入，不开屏即扫。
- **数据 / 文件**收纳在数据 Tab；设置项（环境与权限复查、GNSS、用户管理、导出位置、关于）收纳在设置 Tab。
- **采集运行中**：导航"软锁定"——返回 ≠ 停止；离开运行页时采集 Tab 顶部显示"正在采集"提示条，点击可回；唯一退出 = 长按 2 秒。

> 平台差异见 §6。

## 3. 核心状态机

### 3.1 会话状态 SessionState
```text
Idle ──start()──> Preparing ──全部角色就绪──> Running ──stop()──> Ended
                                   │
                                   └──全局失败(蓝牙关/权限撤销)──> Failed
```

### 3.2 单角色运行态 RoleRuntimeState（每个设备各一份，互相独立）
```text
NotAssigned → Connecting → Subscribing → Collecting
                  ↑              │
                  └──Reconnecting┘（断线，AutoConnect 兜底）
Collecting/Connecting ──不可恢复错误──> Failed(reason)
```

**关键语义**：会话用 supervisorScope，**某个角色 Failed 不取消其它角色**。
- 单设备会话：只有一个角色，"部分失败"退化为"该设备失败"。
- 多设备会话（DUT ≤3 + 参考心率带 ≤1）：某台重连中、其它在采是正常态；单设备为最常见场景。
- 仅当**全局事件**（蓝牙关闭、权限撤销）才整会话进入 Failed。

### 3.3 传感器路状态（DUT 内每路）
```text
Enabled(采样中) ⇄ Paused(运行中临时关) → Streaming(数据在流) → Stalled(单路中断)
```
单路 Stalled 只影响该路落盘，其余路继续。

## 4. 逐屏交互要点

> 中枢 / 数据 / 设置屏见 [v4_android](../prototypes/v4_android.html)（当前）；采集运行 / 异常 / 恢复屏复用 v3 外观（[v3_android](../prototypes/v3_android.html) 仅 legacy 参考）。待补屏清单见 [V4 设计契约 §七](../reviews/BlueTrace_V4_设计契约_2026-06-16.md)。

| 屏 | 关键交互 | 主要状态 |
| --- | --- | --- |
| 启动 | 静默环境检查 + 首启判断 + 可恢复会话 → 决定落地采集 Tab 还是权限页 | 检查中 |
| 采集 Tab（中枢） | 入口卡：设备 / 用户 / 采集模式(Wear⇄Unwear) + 主 CTA「开始采集」；向前推进时隐藏底部 Tab | 就绪 / 待处理 |
| 权限门控 | 仅首启 / 硬权限缺失时弹（门控无返回）；可从设置 Tab 手动复查（带返回）；硬性条件就绪才放行 | 缺项 / 就绪 |
| 设备连接 | 扁平列表：单击连接 / 再点断开；RSSI 过滤；右上连接计数；DUT ≤3 + 参考心率带 ≤1 | 扫描中 / 列表 / 连接中 / 空态 / 连接上限 |
| 用户选择/编辑 | 选择当前用户；新建/编辑别名·性别·生日·身高·体重 | 列表 / 空态 / 编辑 |
| 传感器总控 | 逐路纯开关（开关＝该路是否采集 · 透明传输 · 不控采样率）；不外露高/低频 | — |
| 算法控制 | 勾选设备端算法；下发 | — |
| 数据采集 | 设备卡(含 HR) + 采集类型勾选 + Datas/Time 简单实时数据区 + Start/End 标签；头部 用户·模式·计时 | 就绪 / 采集中 |
| 勾选采集类型 | 清单勾选本次采集哪些路（开关＝是否采集 · 透明传输）；从采集类型芯片进入 | — |
| 长按结束 | 环形进度 2 秒；松手取消 | 进行中 / 完成 |
| 结束摘要 | 会话文件夹构成（原始 HEX 日志 + 解码 CSV + 组合 CSV + manifest）；导出/查看 | — |

## 5. 边界情况清单（每个都要有明确表现，不可静默失败）

| 场景 | 表现 | 行为 |
| --- | --- | --- |
| 采集中连接断开（关蓝牙 / 设备走远 / 通信报错） | 🟡「暂停 · 自动重连中」 | 暂停 → 自动重连；重连后原始 HEX 日志续写续解析，不丢数据、无需用户处理 |
| 坏包 / CRC 错 / 解码失败 / 未知消息类型 / 协议不识别 | 记入**运行日志**并丢弃（不弹 Banner、不打断） | 原始 HEX 始终落盘可重放；无法处理的就丢弃记日志 |
| 存储空间不足（写入失败） | 🔴「已停止」 | 唯一"已停止"语义＝不可继续；安全保存当前数据，引导清理 / 导出 |
| 权限被撤销 | 回权限门控 | 权限是**全局硬门**；停相关能力、引导重授权 |
| 设备扫描无结果 | 空态「未发现设备」 | 重新扫描 / 返回 |
| 连接失败 / 超时 | 设备行内错误 + 重试 | 重试连接 / 选其它设备 |
| 导出失败 / 写入中断 | 红 Banner / 错误 Toast「导出失败」 | 重试导出 / 查看空间 |
| 导出前存储不足（预检） | 红 Banner「存储不足，无法导出」 | 引导清理已导出会话 / 删除旧数据 |
| GNSS 缺定位权限 / 系统定位关闭 | GNSS 设置行内提示 + 引导 | 去授权 / 去系统设置开启 |
| 搜索无结果 | 空态「无匹配会话」 | 清除筛选 / 换关键词 |
| 误触返回键（采集中） | 弹"最小化/停止/取消" | 返回 ≠ 停止 |
| 进程被杀 | 启动时恢复弹层 | 单设备续采；地址失效则定向重扫 |

> **异常模型（V4 精简 · 2026-06-16）**：采集运行只有三态——**采集中 / 暂停·重连中 / 已停止**。连接断开一律暂停 + 自动重连，重连后续写不丢数据；只有"不可继续"（存储满）才"已停止"。坏包 / CRC / 解码失败 / 协议不识别 / 重连**一律进运行日志并丢弃，不弹分级 Banner、不打断采集**。专业采集工具，错误进日志、快速迭代，不做单路降级 / 协议对账 / 逐角色这类发散态。
> **已删除（D-9）**：原"配置带宽超限 → 阻止开始"硬闸取消——App 不配置链路参数（D-5），带宽/丢包仅运行时只读观测，不阻断开始。

## 6. 平台差异（Android ↔ iOS）

> 统一设计语言下，**绝大多数屏两端一致**。以下是**真正分叉**的点——交互上要做平台自适应。组件级映射见 [Design System §平台映射](BlueTrace_Design_System.md#6-平台自适应映射androidios)。

| 维度 | Android | iOS |
| --- | --- | --- |
| 底部导航（V4） | `NavigationBar`（M3，三 Tab：采集/数据/设置，选中药丸指示） | `UITabBar` / `TabView`（SF Symbols，三 Tab，选中 tint） |
| 顶部导航 | 顶部 App Bar，标题居左/居中 | 导航栏：左上返回箭头 + 标题；可用大标题 |
| 返回手势 | 系统返回键 / 手势返回 | **左缘右滑返回**（需保证可滑） |
| 弹层 | Bottom Sheet | **Action Sheet**（破坏性操作红色置底） |
| 轻提示 | Snackbar（底部） | 顶部胶囊 Toast / 无 Snackbar 概念 |
| 主操作 | 底部按钮（**不用 FAB**，与 iOS 对齐） | 底部按钮 / 导航栏右上操作，**不用 FAB** |
| 选择控件 | Material 开关、芯片 | iOS 开关、Segmented Control |
| 安全区 | 状态栏 / 导航栏 inset | 刘海/灵动岛 + 底部 Home Indicator inset |
| 权限弹窗 | 运行时逐项请求 | 一次性系统弹窗 + Info.plist 用途文案 |
| 后台采集 | 前台服务通知常驻 | 无常驻通知；系统态栏蓝条提示后台用蓝牙 |
| 进程恢复 | 启动恢复 + 通知引导 | CBCentralManager 状态恢复 + 启动恢复 |

**交互原则**：流程、状态、信息层级、配色、数据可视化**两端完全一致**；只有"系统级外壳"（导航条、弹层形态、安全区、权限/后台机制）按平台走。这样用户在任一平台都觉得"既是 BlueTrace、又像这台手机该有的样子"。

## 7. 关键交互规则（沉淀为开发统一依据）

1. 用户必须随时知道是否正在采集（运行中全局提示条）。
2. 采集中返回键/手势 ≠ 停止；停止必须明确确认（长按 2 秒）。
3. 设备为**扁平列表**：单击连接 / 再点断开，右上显示连接计数；**DUT ≤3**（第 3 起提示不稳定、第 4 禁止）+ **参考心率带 ≤1**（标准 HRS 0x180D 对照）。至少一台连接成功才允许开始采集；采集单设备优先。
4. 无法解码的数据**记入运行日志并丢弃**；原始 HEX 始终落盘，单点错误不打断采集。
5. 采集运行只有三态：**采集中 / 暂停·重连**（连接断 → 自动重连续写，不丢数据）**/ 已停止**（不可继续才停）；坏包·CRC·解码失败·重连进运行日志，不弹分级 Banner。
6. 设备控制指令即时生效并写入会话 manifest。
7. 采集结束必须有结果摘要；文件必须能被找到和导出。
8. 平台外壳自适应，产品语言统一。
9. 底部三 Tab（采集 / 数据 / 设置）为唯一主 IA；进入子页与采集运行隐藏底部 Tab；v3 向导式仅 legacy 参考。
10. App 不配置 BLE 链路参数（D-5），不阻断开始（D-9）；坏包/CRC/丢包进运行日志、采集页不做观测面板（V4 精简）。用户与 Wear/Unwear 进入一期，写入文件名与 manifest。
