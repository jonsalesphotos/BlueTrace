# 01 · V4 原型 vs 实机实现对比报告

- **原型**：`Docs/prototypes/v4_android.html`（51 屏，11 个板块）
- **实机**：小米 M2101K9C（Redmi K40，1080×2400，MIUI/Android），`io.bluetrace` v1.0.0，2026-07-02 ADB 实拍
- **截图**：`assets/device_20260702/`（22 张，冷启动 → 权限 → 全流程采集 → 数据/设置遍历）
- **走查路径**：冷启动 → 权限门控 → 授权 → 采集主界面 → 设备连接（Mock 扫描）→ 新建/选择用户 → 场景选择 → 在线采集运行 → 采集类型 Sheet → 长按结束 → 结束摘要 → 数据 Tab → 会话详情 → 导出 → 设置 Tab 全部子页 → 离线采集入口

## 结论（TL;DR）

**实现与 V4 原型的一致性非常高**：本轮实拍覆盖的 20+ 屏中，布局、层级、文案、状态色、交互流转基本逐屏对应，多处（离线/在线双 CTA、Default 用户、COMBO_CSV）实现比原型演进得更远。
**当前全部数据链路仍是 Mock**（BT-DUT-042x 假设备 + 0x7E 假格式），与 MILESTONES「M7 被真实协议解码阻塞」一致；**离线采集是占位 Toast**，等协议（本讨论区 03 号文档）落地。

## 发现的差异与缺口

1. **离线采集未实现（预期内）**——唯一的大缺口，阻塞点就是协议（→ 03 号文档）。
2. **GNSS 数据源设置子页缺失**：原型有 GNSS A/B/C 三屏（设置子页，含缺权限/系统定位关闭态）；实机设置 Tab **没有 GNSS 入口**，GNSS 开关被放进运行页「采集类型选择」Sheet 底部。功能不缺（权限门控/环境检查覆盖权限态），但「系统定位总开关关闭」的引导态无处呈现，建议补设置子页或在 Sheet 内加状态行。
3. **采集主界面双 CTA 演进未回写原型**：实机「离线采集 + 在线采集」优于原型「扫描连接 + 开始采集」，方向正确，但 `v4_android.html` 主界面屏与导航链路图应同步，避免设计-实现漂移（原型其余部分同步质量很高，仅此处滞后）。
4. **蓝牙开关权限行的「已开启」环境态**、连接上限第 3 台提示等已按 SPEC 落地（git c457f39/f277f52），本轮未逐台验证多设备上限（Mock 只放 4 台）。
