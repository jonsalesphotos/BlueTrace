# BlueTrace V4 设计决策追踪（Decision Log）

> **这份文档回答"为什么这么做"**：记录 V4 从 v3 演进过程中每个关键决策的**起因、取舍、结论**，供后来者理解"现在为什么长这样"。
> 规格细节（做什么 / 长什么样）见 [V4 设计契约](BlueTrace_V4_设计契约_2026-06-16.md) §九、[REQUIREMENTS](../../REQUIREMENTS.md)（变更记录 v0.10/v0.11）、[原型](../prototypes/v4_android.html)。本文不重复规格，只记**理由**。
> 日期：2026-06-16

---

## 一、起点：一次"找回丢失主界面"的排查

- **诉求**：怀疑从 V2 到现在"丢了主界面"，想找回那份 .html。
- **排查结论**：git 历史完整、**无任何删除**（旧文件只是重命名进 `prototypes/legacy/`）；v3 原型其实**从没画过落地主界面（Home）**，启动直接跳到设备扫描；`Docs/legacy/PIC/` 那组截图是**竞品 Creek ACQ 的参考**，不是 BlueTrace 旧界面。
- **决定**：不存在"可找回的旧主界面文件"，**转向从头设计 V4**。

## 二、关键决策表

| # | 决策 | 起因 / 取舍 | 为什么这么定 |
| --- | --- | --- | --- |
| D-V4-1 | **底部三 Tab（采集/数据/设置）为唯一主 IA** | v3 是向导式中枢，用户要"采集/数据/设置三页 + 底部导航" | 参考竞品 Creek ACQ 的"底部 Tab + 卡片入口"；采集是强状态任务，子页与运行**隐藏 Tab** 防误切 |
| D-V4-2 | **主界面一页放下、字阶收紧** | 用户："字明显太大，明明一页能放下" | 跑设计面板（5 方案 + 3 视角评审），选"HIG 克制"为底：标题 19 / 卡标题 14 / 三级字重 700-600-400；实测内屏 696px 内 0 溢出 |
| D-V4-3 | **设备改扁平列表（取代 DUT/REFERENCE 角色槽 hub）** | 用户："列出所有可连设备，单击连、再点断，想连几个连几个" | 角色槽 hub 属过度设计；扁平列表更直观、天然 1..N。上限 **DUT ≤3**（第 3 起链路不稳定、第 4 禁止）**+ 参考心率带 ≤1**（标准 HRS 0x180D 做对照——后补回） |
| D-V4-4 | **传感器配置=纯开关（去采样率、去高/低频标注）** | 用户："目前透明传输，开关只决定数据要不要上传；高/低频是我俩讨论的内容，不该露给用户" | 数据不因显示"高频"就更快；UI 只表达"采不采这一路"。设备端算法→后期 |
| D-V4-5 | **异常压成三态 + 运行日志**（去红黄蓝分级 / 单路降级 / 协议不匹配 / 配置对账） | 用户："专业采集工具，核心是把数据收回来；无法处理的报错记日志即可，不要发散" | 运行只有 采集中 / 暂停·重连（断连自动重连续写）/ 已停止（不可继续才停）；坏包/CRC/解码失败/重连一律进**运行日志并丢弃**，不打断、不分级 |
| D-V4-6 | **权限=全局硬门**（否定"单路缺权限降级"） | 用户："权限是全局的，不全开根本开不起来采集" | 开始采集前一次性门控；要 GNSS 就开始前授权，否则本次不含 GPS 一路 |
| D-V4-7 | **采集运行照竞品 Data Collection 重做** | 用户："这个问题太多，直接按参考来一版" | 设备卡(含 HR) + 采集类型勾选 + Datas/Time **简单实时数据区** + Start/End 标签；保留**长按 2 秒结束**（防误触）+ 暂停。去波形⇄分包流双视图、只读观测面板、运行中控制面板 |
| D-V4-8 | **受试者 → 用户** | 用户命名偏好 | UI 一律称"用户"（采集用户=被采集者）；工程文档实体仍可用 Subject |
| D-V4-9 | **离线收集 CTA 移除、数据=D-6 会话文件夹** | 一期无服务器（在线/离线属二期）；存储按 D-6 | 一期单一「开始采集」；数据 Tab 列会话文件夹（原始 HEX 日志 + 解码 CSV + 组合包 CSV + manifest），非扁平 rawdata |
| D-V4-10 | **原型为 UI 单一真源；实现文档不复述界面** | 用户："删 UI 实现文本换成'参考 UI 设计 html'，减工作量" | 确立四层分工（见 §四）：实现文档只讲"怎么搭"，界面以原型为准 → 改 UI 只动原型，不再两头同步 |

## 三、被否决 / 降级的方向（避免重走）

- **DUT/REFERENCE 角色槽 hub**（v3+早期 V4）→ 否决，改扁平列表（D-V4-3）。
- **红/黄/蓝三级异常 Banner + 单路降级 + 协议不匹配阻断 + 重连后配置对账**（v3 复用）→ 否决，改三态+运行日志（D-V4-5）。
- **采样率芯片 / 高低频标注 / 打包策略页 / 带宽硬闸**（v3 配置）→ 移除（D-V4-4，对齐 D-5/D-9）。
- **波形 ⇄ 原始分包流双视图 + 只读观测（包速率/丢包/CRC/带宽）+ 运行中控制面板/指令日志** → **降后期**（D-V4-7；对应 F-COL-2/3、F-CTRL-2/4/5 降后期）。
- **协议层"高频批包/低频组帧/分片重组"** → **不是被否决**：仍是链路事实，保留在 [Protocol.md](../architecture/BlueTrace_Protocol.md)；只是 UI 不暴露。

## 四、确立的真源分工（避免文档漂移）

```text
PRD（做什么）─▶ UX_Flows（怎么交互/状态/边界）─▶ Design System + v4_android.html 原型（长什么样）─▶ Architecture / UI_Implementation（怎么搭）
```
- **UI 长什么样** = 原型（唯一视觉真源）；**怎么交互/状态** = UX_Flows；**怎么搭** = 实现文档（指向前两者，不复述界面）。
- 实现文档抬头均有"⚠️ V4 精简 UI 口径"标注作总纲；其内 UI 视觉描述已替换为指向原型。

## 五、过程备注

- 视觉/字阶精修与采集子页构建各跑过一次**多智能体工作流**（方案生成 + 多视角评审），结论已落进原型与契约。
- 期间原型布局曾"左偏"：因 `.grid` 用 `auto-fill`（保留空轨道）→ 改 `auto-fit`（塌缩空轨道、铺满整行）修复。
