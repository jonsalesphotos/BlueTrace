## 二、V4 页面地图

```text
启动
 └─ 静默环境检查（+ 首启判断 + 可恢复会话）
      ├─（首次启动 / 硬权限缺失）→ 权限门控页 PermissionGate（无返回）→ 完成 → 采集 Tab
      └─（否则）───────────────────────────────────────────────→ 采集 Tab
                          （可恢复会话 → 进程恢复弹层 P5，复用 v3）

底部三 Tab（常驻中枢）
├─ ① 采集 Tab（主中枢）
│   ├─ EntryTile 设备  ──→ 设备扫描/分配（搜索 + Service UUID 过滤 + RSSI + 连接 + 刷新）
│   │                        ├─ DUT 扫描   └─ REFERENCE 扫描（标准 HRS 0x180D）
│   ├─ EntryTile 受试者 ──→ 受试者选择 → 受试者编辑（别名/性别/生日/身高/体重）
│   ├─ EntryTile 采集模式（ModeSegment：Wear ⇄ Unwear，内联）
│   ├─（设备连接后）传感器与算法配置 ──→ 传感器总控（开关 + 采样率）+ 设备端算法
│   │                                  ※ 打包策略页已删除（D-5）
│   └─ 主 CTA「开始采集」→ 采集运行（全屏，隐藏 Tab）
│         ├─ 多传感器总览（每路一卡 · 波形 + 实时值 · 设备卡含 HR）
│         ├─ 波形视图 ⇄ 原始分包流（含只读观测：包速率/丢包/CRC/带宽，D-9）
│         ├─ 运行中控制面板（Sheet：不停采集开关传感器/切算法 + 指令日志/ACK）
│         ├─ 时间标签（Start/End Label · 参考 App，F-COL-5 一期可选）
│         └─ 长按 2 秒结束 → 采集结束摘要（→ 数据详情 / 导出）
│
├─ ② 数据 Tab（中枢）
│   ├─ Raw Data File 列表（按日期分组 · Search · 多选/选择态）
│   │      ※ 列表单位 = 会话（文件夹，D-6），非扁平文件
│   └─→ 会话详情（manifest 摘要 + 启用配置 + 文件夹构成 + 选择导出）
│         └─ 导出到公共目录（Android Downloads / iOS Files）→ Toast
│
└─ ③ 设置 Tab（中枢）
    ├─ 环境与权限检查（手动复查，可返回）
    ├─ GNSS 数据源（SettingsRow toggle · 一期正式功能）
    ├─ 受试者管理（列表/CRUD 入口，亦可由采集 Tab 进入）
    ├─ 导出位置 / 存储占用 / 关于
    └─（二期项：服务器/上传/实时透传 → 灰显 + "二期"标注）

横切（任意态）
├─ 采集中 / 暂停·重连 / 已停止 + 运行日志（§九精简 · 取代红黄蓝分级与单路降级）
├─ 进程恢复弹层 — 复用 v3 P5（逐角色：DUT 续采 / REFERENCE 重扫）
└─ "正在采集"全局提示条 — 采集运行中离开时，在采集 Tab 顶部提示可回
```

**导航模型**：底部 Tab 平行切换三中枢；采集是强状态任务，从采集 Tab **向前 push** 逐步推进（设备→受试者/模式→配置→运行→摘要），过程中隐藏 Tab。数据/设置为旁支中枢。

## 六、V4 概念收敛：一期保留 / 二期延后 / 移除

### ✅ 一期保留（需补齐定义）

**受试者 / 采集用户（Subject）**

- **实体**：`Subject { id, alias(如 shb), sex, birthMonth(YYYY-M), heightCm, weightKg, note? }`，本地存储、可多条、可选当前。
- **页面**：受试者选择（列表，空态引导新建）→ 受试者编辑（表单）。入口在采集 Tab 的 EntryTile，亦可从设置进入管理。
- **文件命名**：`<Mode>_<alias>_<yyyyMMdd>_<HHmmss>_<deviceShort>`，如 `Wear_shb_20260521_103014_de54`。
- **manifest 影响**：新增 `subject:{alias,sex,birthMonth,heightCm,weightKg}`。
- **隐私**：建议用别名而非真实姓名；一期不上传、不出设备；"关于/设置"加一句本地存储说明。

**Wear / Unwear（采集模式）**

- **需求来源**：PPG 信号评测需"佩戴 vs 未佩戴"基线对照（Unwear 提供静默/噪声基线）。
- **状态流**：会话级枚举，开始采集前在中枢选定 → 运行头只读显示（`shb · Wear`）→ 写入文件名前缀 + manifest `mode`。
- **数据影响**：数据 Tab 可按 Wear/Unwear 筛选；导出文件名自带前缀便于离线分类。

**GNSS / GPS** — 升为一期正式功能（见 §五）。

### 🔵 二期延后

- **在线采集 / 实时透传 / 服务器上传 / 远程下发 / web open / send log**（参考 App "More" 页的 server config 等全部属此类）。
- **"在线 vs 离线"双 CTA**：二期服务器接入后恢复。
- **Print（打印报告）**：参考 App 有，BlueTrace 用导出替代，不进一期。
- **暗色模式**：二期。
- **`CMD_SET_PACKING` 固件调试控制**：默认隐藏，二期或 debug build。

### ❌ 移除 / 改名

- **"离线收集"CTA**：一期移除（无意义双 CTA）→ 单一「开始采集」。
- **打包策略配置页**：移除（D-5）。
- **带宽硬闸**：移除（D-9）。
- **"ACQ" / "Creek" 竞品命名**：全部去除（"ACQ Project"→"采集模式"）。

## 九、V4 精简修订（2026-06-16 · 应用户反馈收敛）

> 用户复核后明确："专业采集工具，核心是把数据收回来；无法处理的报错记日志即可，不要发散。" 据此对**异常模型**与**设备模型**做一次收敛；原型（`v4_android.html`）与 `UX_Flows` 已同步落地。

**1. 异常模型 → 三态 + 运行日志**
- 采集运行只有 **采集中 / 暂停·重连中 / 已停止** 三态。
- 连接断开（蓝牙关 / 设备走远 / 通信报错）= **暂停 → 自动重连**；重连后原始 HEX 日志 append-only 续写续解析，不丢数据、无需用户处理。
- 只有 **不可继续**（存储满 / 写入失败）才 **已停止**，安全保存后引导清理 / 导出。
- 坏包 / CRC 错 / 解码失败 / 未知消息类型 / 协议不识别 / 重连 **一律进运行日志并丢弃**，不弹分级 Banner、不打断采集。
- **移除**：红/黄/蓝五条合屏、单路降级、参考设备断开提示、协议版本不匹配阻断、重连后配置对账。

**2. 设备模型 → 扁平列表（天然 1..N）**
- 去掉 DUT/REFERENCE 角色槽 hub；改为**一个扁平设备列表**（Service UUID 过滤自己的 DUT + 标准 HRS 0x180D 参考）。
- **单击一行后台连接**（未连接 → 连接中(动画) → 已连接），**再点断开**；右上角显示连接计数。
- 连接上限：**DUT 最多 3 台**（第 3 起标签转黄提示不稳定、**第 4 禁用**）+ **至多 1 个标准心率带参考**（HRS 0x180D 对照用）。
- **采集单设备优先**：多数场景只连一台 DUT；多设备是退化的同一条路径，不喧宾夺主。

**3. 权限 = 全局硬门**
- 权限（蓝牙/扫描/连接；若启用 GNSS 则定位）在**开始采集前一次性门控**；不在采集运行中做"单路缺权限降级"。要 GNSS 就开始前授权，否则本次会话不含 GPS 一路。

**4. 待人工拍板（requirement 级，本轮未改需求基线）**
- ✅ **已对齐（2026-06-16 续）**：`REQUIREMENTS.md` / `PRD` §5 条款（F-DEV-2 扁平 DUT≤3 + 参考心率带≤1、F-CTRL-2/4/5 与 F-COL-2/3 降后期、F-BG-3 三态+运行日志）、§9 追踪矩阵、变更记录 v0.11、背景叙述与用户故事、受试者→用户，均已随 UX_Flows 一并改齐；实现层 Architecture / UI_Implementation / CrossPlatform 已加 V4 精简开发标注（抬头）。**注**：协议层高/低频批包/组帧为链路事实保留。

**5. 采集运行按竞品 Data Collection 重做（2026-06-16 续）**：头部用户·模式·计时 + 设备卡（含实时 HR）+ 采集类型芯片 → **勾选采集类型清单**（开关 = 该路是否采集 · 透明传输 · 不控采样率）+ Datas/Time **简单实时数据区** + Start/End Label + **暂停 + 长按 2 秒结束**。**移除**：传感器采样率与高/低频标注、波形 ⇄ 分包流双视图、只读观测指标（包速率/丢包/CRC/带宽）、运行中控制面板 + 指令日志；设备端算法标「后期」。⚠️ 这使 PRD/UX 的 **F-COL-2/3（双视图）、F-CTRL-4（观测）、F-CTRL-5（运行中控制）、F-CTRL-2（采样率）** 偏离，需人工确认后回灌需求基线。其余文案精简：受试者→用户、删「下一步·配置传感器」（改扫描/停止扫描，扫描 60s 自停）、设备页加 RSSI 过滤滑条、删设备入口传感器标签、GNSS 文案改「采集保存 GNSS 日志」、删「用 XXX 开始」（点行即选中返回）。

## 十、采集运行细化 + 逐组精修 + UX 落档（2026-06-17 续）

> 在 §九 收敛基础上，对**采集运行**做一轮交互细化（粗修），再对全部分组逐屏精修，并把**每屏 UX 交互规格写进原型**。原型 `v4_android.html` 由此从"UI 视觉真源"升级为 **"UI 视觉 + 逐屏交互"双真源**。

### 1. 采集运行交互细化（粗修）

- **「暂停」= 仅暂停数据框滚动显示，不停止接收/采集**（按钮文字仍为「暂停」，语义在注释/文案点明）。数据照常落盘，再点恢复跟随。
- **实时数据区 = 每收到一条写一行 `[时间戳] HEX`**；窗口**锚定底部、无滚动条**（旧行自顶部裁切）；等宽小字号；数据框 **flex 填充**剩余高度。
- **「勾选采集类型」→ 改名「采集类型选择」**（下抽屉 Bottom Sheet）：进入运行A **自动弹出**、右上角可重开、**采集中也不隐藏**入口；圆形勾选保留（用户明确无需改 toggle）。开关 = **该路是否上传（需通信）/落盘**（透明传输、不控采样率）。
- **结束按钮 = 蓝色「结束」**（取代红色「长按 2 秒结束」文案）；长按 2 秒确认仍保留。
- **运行D 长按确认 = 半透明遮罩**叠在采集中界面之上、**透出下面内容**（不整屏遮挡）；中途松手取消。
- **标签区重做**：单行输入框（有文字时**内嵌 X 清空**，去独立"清除"键）+ 右侧 **Pin（瞬时点标签）**；下方 **Start Label ⇄ Stop Label 切换**（长时间区间标签）。
- **横切A** 底部结束按钮统一为蓝色（与运行B 一致）。

### 2. 传感器与算法配置（配置A/B）→ 暂不实现、移至文末

- 两屏**整段物理移动到文档末尾**，标注「暂不实现（已移至文末）」+ 半透明遮罩，设计原样保留备用。
- 采集类型选择已由「采集运行 · 采集类型选择（运行C）」承担；设备端算法为后期能力。

### 3. 逐组精修 + UX 交互落档（每屏 `.screen-ux` 规格块）

- 新增 `.screen-ux` 交互规格组件，**全部 37 屏 100% 覆盖**：逐屏列「点击去向 / 手势 / 状态规则」，把 UX 交互正式写进原型。
- 精修中补齐了一批此前**在用但无样式**的 CSS（渲染异常已修）：`.log-list/.ln`（数据流小字号·三色分级）、`.conn-count`（连接计数徽章）、`.conn-dots`（连接中动画点）、`.list-tile.connected/.disabled/.selected`（行状态）、`.card-title`（卡内标题 14/700）、`.tag.gnss`（并入标签色系）。
- 主界面单一 CTA 注释订正（一期无离线/在线双 CTA）。

### 4. 待回灌需求基线（requirement 级，本轮未改）

- 「暂停=停滚动（不停接收）」语义需在 `UX_Flows` 运行态描述中点明（避免读成"暂停采集"）。
- 采集类型选择改名 / 采集中不隐藏 / 开关=是否上传，属 §九.5 已记偏离项（F-COL-2/3、F-CTRL-2/4/5）的延续，一并在 PRD/UX 回灌时处理。
