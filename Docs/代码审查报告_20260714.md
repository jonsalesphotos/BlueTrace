## 二、KMP 多平台移植能力 —— commonMain 纯净，iOS 目前是"纸面能力"

**结论**：`shared/src/commonMain` **零违规 import**（无 `android.*`/`java.*`/JVM 专属 API；okio 全为多平台 API）、**零 expect/actual**（平台差异一律接口注入：`EpochClock`/`TimeZoneProvider`/`FileSystem`/`SqlDriver`）。但 `shared/build.gradle.kts` 只声明 `jvm()` + `android()`——**没有 iOS target 与 iosMain 源集**，"iOS 可复用"当前是架构纪律而非已验证事实。

### iOS 障碍表

**已隔离**（接口在 shared，iOS 只缺 actual 实现，非结构问题）：`BleClient`（最大件）、`GnssSource`、`StorageMonitor`、`EpochClock`/`TimeZoneProvider`、`EnvironmentRepository`、`AppPreferences`、`SubjectRepository`（只差 iOS SqlDriver）。

**结构性障碍**（具体 Android 类、无 shared 接口、被 VM 直接依赖）：

| 组件 | Android 依赖 | 被依赖处 | 备注 |
| --- | --- | --- | --- |
| `MediaStoreExporter` | Context+MediaStore+java.util.zip | ExportViewModel / SettingsViewModel / DeviceConsoleViewModel | 导出面未抽象（zip 组包历史结论：有意不下沉） |
| `DeviceLogStore` | Context+MediaStore | DeviceConsoleViewModel | 无接口 |
| `ConfigStore` | Context+java.io.File | AppModule / OTA VM | 无接口（本日新增，iOS 时补 `ConfigSource` 接口即可） |
| `OtaZipLoader`/`OtaRunLogStore`/`AppLogMirror`/`BleBackendSwitch` | Context/Uri/File | DEBUG 域 | 随 OTA 工具屏转正再抽象 |
| Path 根派生 | `sessionsRoot(Context)`/`appLogsDir` | AppModule | okio Path 本身多平台，根需 iOS 侧 provider |
| SceneCatalog 加载 | `assets.open("scenes.json")` | AppModule | 解析在 shared ✓，资源读取平台化 |
| ViewModel 层 | androidx.lifecycle | 全部 VM | iOS 需 KMP state-holder；纯逻辑先下沉（见下） |

### 下沉候选（纯逻辑在 app 层，跨平台可单测）

1. 设备行投影（过滤+识别+排序）：`DeviceScanViewModel.kt:70-119` → 抽 shared 纯函数；
2. `ScanDefaults`（纯常量）；
3. `fmtMB` ×3 重复（`MultiOtaViewModel:281`/`OtaTestViewModel:331`/`MultiOtaScreen:465`）+ `nowStamp()` ×2 重复 → shared util 合一（注意去 JVM `String.format`）；
4. `MultiOtaUiState.summaryLine()` 与 `MultiOtaController.summaryLine()` 重复 → 删 app 侧；
5. `SettingsScreens.kt:293` `logClock` 用 java.time → 改调 shared `epochMsToLocalParts`。

### 建议节奏

iOS target 的开启放到二期 M8 前专门一波（加 targets → 修 okio/coroutines 版本矩阵 → BleClient CoreBluetooth actual）；当前保持"commonMain 纪律 + 障碍清单不增长"即可。**设备指令抽象层（W1–W4，见 [`设计/设备指令抽象层_设计.md`](设计/设备指令抽象层_设计.md)）本身就是 KMP 纯度的一次大巩固**——S7 指令面收进 shared profile，App 层 Android 类进一步瘦身。
