二、维护指令实现状态

GET 读设备（0x02）

Key指令状态控制台位置
0x01设备时间✅时间栏
0x04用户信息✅用户信息栏
0x21版本信息（TLV）✅FW/Modem/SecBL/BP
0x22功能掩码✅Func（真机回 0，逐位待固件）
0x24电量✅Battery
0x26身份 SN/MAC/IMEI/ICCID✅设备信息栏（MAC 反序修正）
0x28绑定状态✅Bond
0x02/03/05..15/25/27/2A闹钟/目标/佩戴/单位/久坐/监测开关/提醒/表盘…◻业务类

SET 写设备（0x03）

Key指令状态控制台位置
0x01时间同步（对时）✅时间栏 · 同步时间
0x04用户信息（可编辑写入）✅用户信息栏 · 编辑并写入
其余 SET目标/提醒/监测/表盘…◻业务类

DEV_CTRL 设备控制（0x07）

Key指令状态控制台位置
0x01关机✅危险操作（断链判据）
0x02重启✅危险操作
0x03恢复出厂✅危险操作
0x04 / 0x05找手表 / 结束找表✅危险操作 · toggle 可停止
0x07 / 0x09拉取设备日志 / 回传块✅设备日志 → Download/BlueTrace/logs/
0x08实时 BLE 日志⛔ 固件 #if 0—
0x0A / 0x0B拉取/回传算法特征文件⬜ 后续诊断增强
0x06找手机结束◻手机侧，非维护

IND 设备→App（0x05）

Key指令状态
0x0C心跳（电量/seq）✅ 被动解析
其余来电/运动/音乐/相机/健康告警…◻ 业务类

三、除 OTA 外未实现的维护相关指令

指令族说明状态 / 为何暂缓
BOND 绑定族（0x01）业务绑定握手 + 鉴权码 KEYCODE无需 真机实证维护指令免绑定直发；固件不校验 KEYCODE。仅当未来加门控才需要
FILE_TRANS / OTA（0x0F 全族）固件/资源 OTA 升级⬜ 二期 端到端编排（文件清单/类型/生效触发）文档缺失，需固件组补升级包规范
FILE_ALGO（0x0A/0x0B）拉取算法特征文件⬜ 后续 诊断增强，复用日志拉取通道模式即可加
GET_DIAL / 表盘管理表盘信息/切换/排序/删除◻ 业务类，非维护工具
TEST 工厂模式（0x08）进厂模式 / 马达开关◻ 产测专用；MOTO_ON/OFF 固件无 handler
一句话结论：维护控制台需要的读写与控制指令已全部实现并真机验证（设备信息全套 GET、时间/用户 SET、找手表/重启/关机/恢复出厂、日志拉取、心跳）。 除 OTA（二期）外，其余未实现项要么是业务类手表功能（不属维护工具），要么固件侧 #if 0 未开（BLE_LOG），要么可选诊断增强（算法特征文件）——均不阻塞控制台运维用途。
